It took 8.4 seconds to read the file.  Let’s read it again now that the file should be in the filesystem cache and see how long it takes now.

Boom!  It took only .2 seconds compared to 8.4 seconds to read it when the file was not cached.  To verify let’s repeat this again by first clearing the cache
and then reading the file 2 times.

It worked perfectly as expected.  8.5 seconds for the non-cached read and .2 seconds for the cached read.

Conclusion

The page cache is automatically enabled on Linux systems and will transparently make IO faster by storing recently used data in the cache.  If you want to
manually clear the cache that can be done easily by sending an echo command to the /proc filesystem indicating to the kernel to drop the cache and free
the memory used for the cache.  The instructions for running the command were shown above in this article and the experimental validation of the cache
behavior before and after flushing were also shown.
